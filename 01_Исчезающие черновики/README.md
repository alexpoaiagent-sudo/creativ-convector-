# Исчезающие черновики

## Назначение

Промежуточное хранилище для обработанных заметок с метаданными. Черновики требуют доработки перед переносом в приоритетные проекты.

## Структура черновика

Каждый черновик содержит YAML front matter с метаданными:

```yaml
---
created: 2026-01-14
updated: 2026-01-14
role: F5
project: ВК кофе (сеть кофеен)
status: черновик
---
```

## Метаданные

- **created:** Дата создания черновика
- **updated:** Дата последнего обновления (когда добавлены новые заметки)
- **role:** Роль из таблицы 3×3 (F1-F9)
- **project:** Название проекта или "нет проекта"
- **status:** Всегда "черновик" в этой папке

## Workflow

1. **Создание:** AI агент создает черновик из заметок
2. **Пополнение:** Новые заметки добавляются в существующий черновик
3. **Доработка:** Вы дорабатываете черновик
4. **Перенос:** Готовый черновик становится "заготовкой" в проекте

## Шаблон

Используйте файл `_template.md` для создания новых черновиков вручную.

## Важно

- Черновики могут пополняться новыми заметками
- Дата `updated` обновляется при каждом пополнении
- Черновики без проекта помечаются как "нет проекта"
