# Инструкция по использованию системы управления заметками

## Обзор

Эта система автоматизирует работу с заметками в Obsidian, используя AI агента для распределения заметок по проектам и синхронизации с базой знаний.

## Структура проекта

```
/Users/alexander/Documents/Заметки/
├── Изчезающие заметки/          # Временные заметки для обработки
├── Черновики по приоритетным проектам/
│   ├── БК - Кофейня/            # Заметки по кофейне
│   ├── ВК - Проект/             # Заметки по ВК
│   ├── Марафон/                 # Заметки по марафону
│   ├── Стратег/                 # Стратегические заметки
│   └── Общее/                   # Общие заметки
├── scripts/                     # Скрипты автоматизации
├── reports/                     # Отчеты
└── config.json                  # Конфигурация
```

## Workflow

### 1. Создание заметок

Создавайте новые заметки в папке **"Изчезающие заметки"**. AI агент автоматически обработает их и распределит по проектам.

### 2. Обработка заметок

Запустите обработку заметок:

```bash
cd "/Users/alexander/Documents/Заметки"
npm run process
```

Или напрямую:

```bash
node scripts/process-notes.js
```

**Что делает скрипт:**
- Сканирует папку "Изчезающие заметки"
- Анализирует содержимое каждой заметки
- Определяет проект на основе ключевых слов
- Перемещает заметку в соответствующую папку
- Создает отчет о перемещении

### 3. Синхронизация с базой знаний

Для синхронизации с репозиторием "ВК кофе":

```bash
npm run sync
```

Или:

```bash
node scripts/sync-knowledge-base.js
```

**Что делает скрипт:**
- Анализирует черновики по кофейне
- Ищет соответствия в базе знаний
- Определяет роли (бариста, сомелье, менеджер)
- Создает отчет о возможных местах для интеграции

### 4. Генерация отчета

Для получения общего статуса системы:

```bash
npm run report
```

Или:

```bash
node scripts/generate-report.js
```

## Конфигурация

Отредактируйте `config.json` для настройки путей к репозиториям:

```json
{
  "repositories": {
    "notes": {
      "name": "Стратег",
      "path": "../../../Стратег",
      "remote": "https://github.com/your-username/strateg.git"
    },
    "knowledgeBase": {
      "name": "ВК кофе",
      "path": "../../../VK-offee",
      "remote": "https://github.com/your-username/vk-coffee.git"
    }
  }
}
```

## Git workflow

### Фиксация изменений

После обработки заметок зафиксируйте изменения:

```bash
cd "/Users/alexander/Documents/Заметки"
git add -A
git commit -m "Обновление заметок"
```

### Создание GitHub репозитория

1. Создайте репозиторий на GitHub
2. Добавьте remote:

```bash
git remote add origin https://github.com/your-username/notes.git
git branch -M main
git push -u origin main
```

### Синхронизация с GitHub

```bash
git pull origin main  # Получить изменения
git push origin main  # Отправить изменения
```

## Автоматизация

### Создание cron задачи (macOS/Linux)

Для автоматической обработки заметок каждый день:

```bash
crontab -e
```

Добавьте строку:

```
0 9 * * * cd /Users/alexander/Documents/Заметки && node scripts/process-notes.js
```

### Использование с Obsidian Shell Commands

1. Установите плагин "Shell Commands" в Obsidian
2. Добавьте команды:
   - Обработать заметки: `node scripts/process-notes.js`
   - Синхронизация: `node scripts/sync-knowledge-base.js`
   - Отчет: `node scripts/generate-report.js`

## Отчеты

Все отчеты сохраняются в папке `reports/`:

- `processing-report.md` - отчет о обработке заметок
- `sync-report.md` - отчет о синхронизации с базой знаний
- `status-report.md` - общий статус системы

## Troubleshooting

### Заметки не обрабатываются

Проверьте:
1. Заметки находятся в папке "Изчезающие заметки"
2. Файлы имеют расширение `.md`
3. Файлы не являются системными (не начинаются с точки)

### Ошибка при синхронизации

Проверьте:
1. Путь к репозиторию "ВК кофе" в `config.json`
2. Репозиторий существует и доступен
3. Есть права на чтение файлов

### Скрипты не запускаются

Убедитесь, что:
1. Node.js установлен: `node --version`
2. Скрипты исполняемые: `chmod +x scripts/*.js`
3. Вы находитесь в правильной директории

## Следующие шаги

1. Настройте GitHub репозитории для "Стратег" и "ВК кофе"
2. Обновите пути в `config.json`
3. Настройте автоматическую обработку через cron или Obsidian
4. Регулярно проверяйте отчеты
5. Синхронизируйте изменения с GitHub
