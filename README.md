# Creative Convector — Творческий конвейер

Obsidian-хранилище заметок с AI-агентом для автоматического распределения по методологии SRT/FPF.

## Как это работает

```
Телефон/Obsidian          GitHub Actions             Obsidian
      │                        │                        │
      ▼                        ▼                        ▼
1. Исчезающие заметки  →  AI агент анализирует  →  2. Черновики/[Проект]
   (пишешь заметки)       и распределяет           (готовы к доработке)
                               │
                               ▼
                          System/Отчёты AI
                          (отчёт + оригиналы)
```

1. Пишешь заметки в `1. Исчезающие заметки/` (с телефона или Obsidian)
2. Push в GitHub → автоматически запускается AI агент
3. Агент анализирует каждую заметку, определяет проект и роль (F1-F9)
4. Заметка перемещается в `2. Черновики/[Проект]/` с frontmatter
5. Оригинал сохраняется в `System/Обработанные/`
6. Создаётся отчёт в `System/Отчёты AI/`

## Методология SRT (Systems-Roles-Table)

AI агент использует матрицу 3x3 для классификации заметок:

```
                     Предприниматель    Инженер           Менеджер
                     (Зачем?)          (Как устроено?)   (Как работает?)
┌────────────────────────────────────────────────────────────────────┐
│ Надсистема        F1-Контекст       F2-Окружение      F3-Взаимодействие │
│                   рынок, тренды     интеграции        партнёры, клиенты │
├────────────────────────────────────────────────────────────────────┤
│ Целевая система   F4-Требования     F5-Архитектура    F6-Реализация     │
│                   идеи, задачи      структура         процессы          │
├────────────────────────────────────────────────────────────────────┤
│ Конструктор       F7-Принципы       F8-Платформа      F9-Команда        │
│                   стратегия         инструменты       люди, роли        │
└────────────────────────────────────────────────────────────────────┘
```

Каждая заметка получает:
- **Проект** — к какой целевой системе относится
- **Роль F1-F9** — какую ячейку матрицы заполняет
- **Frontmatter** — метаданные для Obsidian

## Автоматизация

### GitHub Actions workflow

Триггер: push в `1. Исчезающие заметки/` на ветку main

Файл: `.github/workflows/distribute-notes.yml`

Что делает:
1. Находит `.md` файлы в `1. Исчезающие заметки/`
2. Запускает `.github/scripts/distribute_notes.py`
3. Коммитит результат обратно в репозиторий

Также можно запустить вручную: GitHub → Actions → AI Agent - Distribute Notes → Run workflow

### Скрипт распределения

Файл: `.github/scripts/distribute_notes.py`

Что делает:
1. Читает каждую заметку
2. Убирает старый frontmatter и артефакты ChatGPT
3. Анализирует содержимое по ключевым словам
4. Определяет проект и роль F1-F9
5. Добавляет SRT frontmatter
6. Перемещает в `2. Черновики/[Проект]/`
7. Сохраняет оригинал в `System/Обработанные/`
8. Создаёт отчёт
